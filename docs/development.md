# 开发约定

## 当前阶段

只做项目整理，不写功能实现。

## 基本约定

- 默认中文文档。
- 命令、路径、状态值使用反引号。
- 后端使用 `Node.js + TypeScript`。
- 后端服务端口从 `5301` 开始。
- 第一阶段只保留 3 个独立运行服务：`identity-service`、`camp-service`、`notification-service`。
- 第一阶段不用 Docker Compose，本地依赖直接安装到开发机。
- 先更新文档，再写实现。
- 先确认目标和收益，再进入技术方案。

## 后续实现前必须补齐

- 数据库方案。
- 鉴权方式。
- 文件存储方案。
- API 规范。
- 测试策略。

## 不建议现在做

- ⚠️ 技术过度设计：服务注册、链路追踪、复杂网关。
- ⚠️ 技术过度设计：第一阶段不引入 Docker Compose 或容器编排。
- ⚠️ 时间黑洞：完整规则引擎。
- ⚠️ 伪需求：没有真实使用场景前做复杂报表。
