# 端侧划分

## 该不该继续拆端

不该。

## 为什么

当前业务还在 MVP 阶段，端太多会增加路由、权限、部署、菜单和验收成本。4 个端已经覆盖核心角色和主流程。

## 端清单

| 端 | 目录 | 职责 |
| --- | --- | --- |
| 学员端 | `apps/student-web` | 看任务、提交作业、查看反馈、重提。 |
| 助教端 | `apps/assistant-web` | 批改、评论、退回、评分、查看待处理任务。 |
| 讲师端 | `apps/instructor-web` | 发布作业、查看班级完成情况、看成绩与统计。 |
| 管理后台 | `apps/admin-web` | 课程、班级、角色、时间规则、评分配置管理。 |

## 合并规则

- 课程管理、班级管理、角色管理、时间规则、评分配置全部归到 `admin-web`。
- 作业发布归到 `instructor-web`，不单独做作业管理端。
- 成绩统计归到 `instructor-web`，不单独做统计端。
- 学员作业历史和反馈归到 `student-web`，不单独做个人中心端。
- 助教待处理、批改、退回归到 `assistant-web`，不单独做审核端。

## 暂不做

- ⚠️ 技术过度设计：不拆课程端、统计端、配置端。
- ⚠️ 收益不匹配：不为单一功能创建独立前端应用。
