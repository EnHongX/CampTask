# 业务流程

## 作业主流程

```text
讲师发布任务
  -> 学员查看任务
  -> 学员提交作业
  -> 助教批改
  -> 通过：生成成绩和反馈
  -> 退回：学员修改后重提
  -> 讲师查看统计
```

## 状态建议

### 作业状态

- `draft`：草稿。
- `published`：已发布。
- `closed`：已截止。
- `archived`：已归档。

### 提交状态

- `not_submitted`：未提交。
- `submitted`：已提交；重提后仍使用该状态，用版本号区分第几次提交。
- `reviewing`：批改中。
- `returned`：已退回。
- `accepted`：已通过。

### 批改结论

- `approved`：通过。
- `returned`：退回修改。

## 版本规则

- 每次提交生成一个提交版本。
- 退回后重提不创建新的作业，只创建新的提交版本。
- 批改反馈绑定到具体提交版本。
- 统计默认以最新有效提交版本为准。

## 当前不展开

- 补交审批流程。
- 多助教协同批改分配。
- 申诉和复核流程。
- 自动查重。
- 自动评分。
